voting website - database schema V1

~<...> means key value
<...>? means optional value


Vote
(~voterID, ~choiceID, amount, mostRecent, comment?)

Topic
(~id, name, description, startTime, endTime, startCondition?, endCondition?) // set last two to null at first

Choice
(~name, ~topicID, description?, proposedBy?, dateProposed?)

UserVote
(~userID, ~topicID, votesAllocated)

User
(~id, username, dateJoined, email?)

EndCondition 
(~endConditionID, description?, endCode)

NotifyUser
(~userID, ~topicID, message?)
